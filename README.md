# Blindly — Mobile UI Kit

**Blindly** is a modern, clean, non-cringe dark-themed UI for a mobile-first dating app where users first discover each other via hobbies and personality. Photos remain hidden until a mutual unlock occurs.

## Tech Stack

- **Framework**: React Native (Expo Router)
- **Language**: TypeScript
- **Styling**: TailwindCSS (via NativeWind v4)
- **State Management**: Zustand
- **Icons**: Lucide React Native

- `p-2` = 8px

## Component Library

Located in `expo/components/ui`.

### 1. Button
Primary call-to-action component.
```tsx
<Button variant="primary" size="lg" onPress={...}>
  Get Started
</Button>
```
- **Variants**: `primary`, `secondary`, `ghost`, `danger`
- **Sizes**: `sm`, `md` (default), `lg`

### 2. Card
Container for content.
```tsx
<Card variant="elevated" padding="md">
  <Content />
</Card>
```
- **Variants**: `default` (surface), `elevated` (lighter), `outlined`

### 3. Chip
Used for hobbies and interests.
```tsx
<Chip label="Hiking" variant="primary" selected={true} />
```
- **Variants**: `default`, `primary` (selected), `outline`, `ai`

### 4. Avatar
Handles user images and the "Locked" state.
```tsx
<Avatar source={uri} locked={true} size="lg" />
```
- **Props**: `locked` (bool) replaces image with lock icon.

### 5. Badge
Small status indicators or trait tags.
```tsx
<Badge label="95% Match" variant="ai" />
```

### 6. LikertScale
5-point agreement scale for personality questions.
```tsx
<LikertScale question="I am spontaneous." value={3} onChange={setValue} />
```

## UX Rules & Patterns

1.  **The Reveal Logic**:
    - Photos are **never** shown in the Discovery feed.
    - Users see: Bio, Age, Distance, Hobbies, Personality Traits.
    - Unlock requires mutual consent via the "Unlock Request" flow.

2.  **AI Transparency**:
    - Any content generated by AI (bios, reply suggestions) must be labeled.
    - Use the `Sparkles` icon and `#FFD166` (Yellow) color for AI indicators.

3.  **Safety First**:
    - Reporting and blocking must be accessible from Profile and Chat.
    - Unlock modals must remind users about safety.

4.  **Rating System**:
    - Post-reveal ratings are 1-10.
    - **7** is the threshold for a "High Match".
    - Ratings help train the recommendation algorithm.

## Getting Started

1.  **Install Dependencies**:
    ```bash
    cd expo
    npm install
    ```

2.  **Run the App**:
    ```bash
    npx expo start
    ```

3.  **Run Mock API** (Optional):
    ```bash
    cd services
    go run main.go
    ```
